# SEE 生命印迹 H5 思维画像报告系统 MVP 说明

## 1. 项目目标

做一个微信 H5 报告生成系统。用户通过二维码进入 H5，上传一张固定格式的“我的思维画像”思维导图照片，系统用 AI 自动识别图中的填写内容，生成 3 份个性化图文报告。

第一阶段先做本地测试版，不部署，不接支付，重点验证：

- AI 是否能准确识别 25 题 ABCD 连线结果
- AI 是否能识别右侧手写字段
- 是否能基于识别结果生成 3 份报告
- 单个用户 3 份报告 AI 成本是否能控制在 0.5 元以内

## 2. 输入图片说明

用户上传的是固定格式的思维导图照片。

图片中有三类内容：

1. 固定打印内容
   - 模板中的问题、选项、模块标题、故事文字都是固定的。
   - 系统不需要每次识别这些内容。

2. 25 题连线结果
   - 左侧有 25 个固定问题。
   - 每题通过绿色连线连接到右侧 A/B/C/D 选项。
   - 系统只需要判断每题对应 A/B/C/D 哪个结果。

3. 右侧手写字段
   - 图片右侧有用户填写或书写的部分。
   - 示例字段包括：“我”“深度”“结果”“分析”。
   - 这些属于动态字段，需要 AI 识别。

## 4. 识别策略

优先使用 AI 视觉模型识别。

用户体验：

```text
上传图片
→ AI 自动识别 25 题 + 手写字段
→ 如果识别可信度高，直接生成报告
→ 如果识别可信度低，只让用户确认不确定项
→ 生成报告
```

不希望用户每次都手动确认。只有识别不清晰时，才弹出确认按钮或确认页。

可信度规则可参考：

- 必须识别出 25 个题目
- 每题只能是 A/B/C/D
- 必须识别出关键手写字段
- 平均置信度大于等于 0.85 可直接生成
- 低置信度字段进入确认页
- 识别失败不扣额度

## 5. 报告生成逻辑

每个用户最多生成 3 份报告。

三份报告不是三次独立分析，而是基于同一次识别结果：

1. 第 1 份：基础思维画像报告
   - 包含用户的核心思维模式、大脑战略偏好、接收器、输出方式、优势、风险、成长建议。

2. 第 2 份：沟通/关系报告
   - 基于第一份画像底稿，延展成家庭、亲子、伴侣或团队沟通建议。

3. 第 3 份：成长行动报告
   - 基于第一份画像底稿，生成行动计划、成长路径、练习建议。

第一份报告会生成并保存一个“用户画像底稿”，后两份复用这个底稿，不重新识图，不重新全量分析，以降低成本。

## 6. 成本目标

目标：一个用户 3 份报告总 AI 成本控制在 0.5 元以内。

推荐成本结构：

```text
第 1 份：图片识别 + 结构化画像 + 基础报告，约 0.15-0.25 元
第 2 份：基于画像底稿生成延展报告，约 0.05-0.10 元
第 3 份：基于画像底稿生成行动报告，约 0.05-0.10 元

合计目标：0.25-0.45 元
预算上限：0.5 元 / 用户
```

推荐优先测试国产模型：

- 通义千问 / Qwen VL：图片识别
- 豆包视觉模型：图片识别备选
- DeepSeek Chat：文本报告生成
- Qwen Flash / Plus：报告生成与结构化输出

OpenAI 可作为高质量备用方案，但成本较高。

## 7. 权益和二维码设计

正式版需要支持二维码权益控制。

一个二维码可以绑定一组权益，例如：

```json
{
  "code": "SEE20260611A",
  "type": "group_invite",
  "max_users": 10,
  "reports_per_user": 3,
  "total_reports": 30,
  "used_users": 0,
  "used_reports": 0,
  "expires_at": "2026-12-31"
}
```

用户微信扫码进入 H5 后：

```text
微信扫码
→ 打开 H5
→ 微信授权或手机号登录
→ 系统识别二维码 code
→ 判断该二维码是否还有名额
→ 给该用户绑定 3 份报告额度
→ 用户生成报告
```

规则：

- 一个二维码最多允许 10 个用户
- 每个用户最多 3 份报告
- 总计最多 30 份报告
- 同一个微信用户重复扫码不重复占名额
- 超过 10 人或 30 份后提示额度已用完
- 报告生成成功后才扣额度
- 上传失败、识别失败不扣额度

## 8. 兑换码设计

正式版也支持兑换码购买。

两类码：

1. 单人兑换码
   - 兑换 1 个用户的 3 份报告额度。

2. 团体二维码
   - 允许 10 人，每人 3 份，总计 30 份。

第一阶段可以不接支付，人工收款后后台发兑换码或二维码。

## 9. 测试版范围

当前阶段先做本地测试版，不部署，不接微信。

本地测试版功能：

- 上传思维导图图片
- AI 自动识别 25 题 ABCD
- AI 识别右侧手写字段
- 输出结构化 JSON
- 低置信度时显示确认项
- 生成 3 份报告
- 显示每次 token 和成本估算
- 模拟二维码额度：10 人 / 每人 3 份 / 总计 30 份
- 本地保存报告记录即可

暂不做：

- 真实微信登录
- 真实二维码访问
- 服务器部署
- 微信支付
- 正式后台
- ICP 备案
- 多经销商复杂权限

## 10. 正式上线所需系统

正式上线时需要：

- 域名
- HTTPS
- 阿里云服务器
- 数据库
- 对象存储 OSS
- 微信公众号网页授权
- AI 模型 API Key
- 后台管理系统
- 二维码/兑换码管理
- 用户额度管理
- 报告记录管理
- token 成本统计

如果服务器放中国大陆，还需要 ICP 备案。

## 11. 建议开发顺序

1. 定义 25 题标准模板和 A/B/C/D 含义
2. 定义右侧手写字段
3. 做 AI 识图 prompt，输出固定 JSON
4. 做本地上传页面
5. 做识别结果校验和低置信度确认
6. 做第一份基础报告
7. 保存用户画像底稿
8. 基于底稿生成第二、第三份报告
9. 记录 token 和成本
10. 做本地模拟二维码额度
11. 测试 10-30 张真实拍照图片
12. 评估识别准确率和报告质量
13. 再决定是否部署正式版

## 12. 核心产品原则

这个系统不是通用 OCR，也不是通用思维导图识别。

它是一个“固定模板识别 + 个性化报告生成系统”。

AI 只负责识别变量和生成表达；固定模板、知识库、题目解释、权限控制，都应该由系统管理。
