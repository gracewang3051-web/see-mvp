# 07 Report Framework

## 1. 定位

Report Framework 负责将先天优势解码结果组织成可交付报告。

当前版本所有报告必须遵循：

```text
行为解码
↓
特质地图
↓
场景影响
↓
支持方案
↓
行动计划
```

而不是：

```text
指标罗列
↓
定义解释
↓
泛泛建议
```

---

# 2. 通用报告结构

```markdown
# SEE 先天优势解码报告

## 1. 核心结论：行为解码

用一段话回答：
这个人当前表现出的行为，从先天优势角度如何理解？

## 2. 先天特质地图

用结构化方式呈现：
- 核心优势
- 主导通道
- 行为模式
- 功能区强项
- 成长提醒区

## 3. 场景表现分析

说明这些先天特质在具体场景中如何表现。

## 4. 优势发挥方式

说明什么样的环境、任务、关系会帮助其发挥。

## 5. 潜在消耗点

说明什么样的环境可能造成消耗，但不能写成缺陷。

## 6. 支持方案

给家长、老师、伴侣、管理者或本人可执行建议。

## 7. 行动清单

输出短期可执行动作。

## 8. 边界说明

说明本报告基于先天优势测评，不用于医学、心理诊断或命运预测。
```

## 11. 报告质量标准

1. 开头必须先讲“行为解码”，不能先堆指标。
2. 每个结论必须有数据来源。
3. 每个风险必须转化为支持策略。
4. 低分区只写“成长提醒”或“支持需求”，不写缺陷。
5. 儿童报告以家长可执行为导向。
6. 企业报告避免评价员工人格，改写为岗位适配与组织配置。
